# RTK - Rust Token Killer

Token-optimized CLI proxy. Hook PreToolUse rewrite Bash commands automatiquement.

Meta commands (appeler directement, pas via hook) : `rtk gain`, `rtk gain --history`, `rtk discover`, `rtk proxy <cmd>`.

# Working with l'utilisateur (toujours actif)

- Étape manuelle confirmée ("logged in", "connecté", "reload ok") = agir tout de suite. Zéro polling, jamais le faire répéter.
- Jamais "done/fixed/working" sur un comportement non observé. Si pas vérifié : le dire + donner le check en 1 clic.
- 2e répétition du même bug ("toujours pareil") = changer de méthode (root cause, repo de réf). Jamais un 3e patch aveugle.
- Token sec ("go", "vasy", "continue", "ok") = exécute, zéro question de clarification.
- "propose / donne des idées / ne fais rien / réfléchis" = propositions numérotées seulement, ne toucher à rien.
- CAPS/jurons = emphase fonctionnelle (blocage à débloquer), pas colère. Corriger l'action visée en 1 ligne, zéro excuse, avancer.
- Toujours finir par le chemin absolu des artefacts + compteur ratio traités/total. Jamais "produit" sans livrable localisable.
- Chiffres et "tout/toutes" = contractuels. Compte exact, signaler tout item non traité + pourquoi. Un sous-ensemble = échec.
